# 自然な日本語を書く

厳密な規範集ではなく、「AIが書いた感じ」を消すための要点集。
迷ったら、信頼できる同僚に口頭で説明するときの言い方に寄せる。

## 原則

- 主張はそのまま書く。前置きや総括で包まない。
- 一つのことは一度だけ言う。言い換えて繰り返さない。
- 強調(太字・記号・修辞)に頼らず、文の順序と具体性で伝える。
- 硬い漢語や翻訳調より、普通の話し言葉に近い書き言葉を選ぶ。

## AIっぽい型を避ける

次の表現は、論点を増やさず「ちゃんと書いている感」だけを足す。見つけたら削るか、中身のある文に置き換える。

- 予告と総括:「重要なのは〜です」「本記事では〜を見ていきます」「まとめると」「要するに」(直前の言い換えだけのとき)
- 空虚な形容:「不可欠」「鍵となる」「包括的」「多角的」「根本的な」(中身を説明せず強調だけする)
- 空虚な動詞:「掘り下げる」「深掘りする」「言語化する」(何をどうしたのか書かずに終わる)
- 中身のない強調:「非常に」「極めて」「大いに」
- 接続の連打:「さらに」「また」「加えて」を段落ごとに置かない
- 対句の決め台詞の多用:「AではなくBです」「単なるAではありません」
- ダッシュ(— や ――)を地の文で使わない。括弧で挿入するか、句点で文を分ける

構成にも同じことが言える。箇条書き・見出し・太字・表を乱発しない。
2〜3文で済む話は普通の文章で書き、見出しは3階層以内にとどめる。

## 自然でやわらかくするコツ

- 翻訳調をほどく:「〜することができます」→「〜できます」、「〜する必要があります」→「〜が必要です」「〜してください」
- 過剰な敬語を使わない:「〜いたします」「〜でございます」より、です・ます調の丁寧さで十分
- 受け身より行為者を主語に:「変更が適用されました」→「変更を適用しました」
- 確かなことは言い切り、不確かなことは「〜のはず」「〜かもしれない」を残す。機械的にどちらかへ倒さない
- 文の長さを揃えすぎない。短い文と長い文が混ざるほうが読みやすい
- 体言止めと感嘆符は要所だけに使う
- 文体(です・ます / だ・である)は一つの文書内で統一する

書き換えの例:

> 悪い例: 本設定は開発体験の向上において極めて重要な役割を果たします。まず、キャッシュを有効化することができます。さらに、ビルド時間を大幅に短縮することが可能です。
>
> 良い例: この設定を入れると、2回目からはキャッシュを使うのでビルドが速くなります。

## 書き上げたら

- 読み上げるつもりで見直し、引っかかる箇所を直す。声に出して不自然な文は、書き言葉としても硬い
- 削っても意味が変わらない文・形容を削る
- 段落の内容を一文で言えないなら、段落を分ける
